HTML5에서 요소의 변화 : 읽어보세요

  • 새롭게 추가된 요소/속성, 지원이 중단된 요소/속성
  • 적용 의미가 변경된 요소 → b, i, cite, hr, address, s, a, u, strong, small

물리적 스타일 태그
<b> : 진하게 표시 // • 적절한 다른 요소가 없을 때만 사용하는 최후의 수단
<i> : 기울어진 형태(이탤릭체)로 표시 // • 적절한 시멘틱 요소가 없을 경우에 한해서 사용
<s> : 글자 가운데를 통과하는 줄(취소선)을 표시 // • 더 이상 정확하지 않거나 관련이 없는 텍스트
<u> : 밑줄을 표시 // 철자가 틀린 단어나 중국어의 고유명사와 같이 일반 텍스트와 스타일이 다른 텍스트 // • 하이퍼링크와의 혼란 가능성→ 사용 자제
<small> : 기본 글자의 크기보다 작은 크기의 글자로 표시 // 작게 표시해야 하는 부수적 해설이나 이용 조건 또는 법적 공지 등
<sup> : 글자를위첨자로 표시
<sub> : 글자를아래첨자로 표시

논리적 스타일 태그
<cite> : 책, 시, 그림, 영화 등의 작품 제목을 표시 // • 이탤릭체, 인용
<strong> : 중요한 내용을 표시 // • 중요한 텍스트
<address> :
서울시 종로구 // • body 요소 내에서 사용되는 경우 →문서 저자의 정보 // • article 요소 내에서 사용되는 경우→ 해당 기사의 저자 정보

<code> : 컴퓨터 코드를 표시
<samp> : 프로그램의 샘플 출력을 표시
<var> : 변수를 표시
<dfn> : 용어에 대한 정의를 표시
<em> : 강조하는 내용을 표시
<abbr> : 축약형을 표시
<kbd> : 키보드로 입력한 내용임을 표시
<q> : 짧은 인용구를 표시

그외
<hr> : 수평선 → 페이지 내에서 주제가 변경되어 콘텐츠를 구분하는 의미론적 용도
<a> : 하이퍼링크, 앵커 → 항상 하이퍼링크 또는 하이퍼링크 표시자
• name 속성은 id 속성으로 대체
• article 요소 내에서 사용되는 경우→ 해당 기사의 저자 정보
• 속성 추가→ download, media 등
