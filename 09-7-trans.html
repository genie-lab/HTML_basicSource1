transition 전환


transition 전이, 어떤 이벤트에 대해서 지정한 시간 내에 속성의 특정 상태가 다른 상태로 부드럽게 변화하는 것
스타일의 점진적인 변화 → 애니메이션 효과 부여
속성
transition-property 전환 효과를 부여할 속성의 이름 지정
transition-duration 전환이 진행되는 시간 지정 (0초)
transition-timing-function 전환이 진행되는 속도의 형태 지정
transition-delay 전환이 시작되기 전의 지연 시간 지정
transition 전환 관련 속성의 일괄 지정

마우스를 올려놓으면 사각형의 폭이 늘어남
느리게 시작해서 점점 빠르게 진행하다가 후반부에서 다시 느리게 진행 transition-timing-function: linear
처음부터 끝까지 일정한 속도로 진행 transition-timing-function: ease
느리게 시작해서 점점 빠르게 진행 transition-timing-function: ease-in
빠르게 시작해서 점점 느리게 진행 transition-timing-function: ease-out
느리게 시작해서 중간까지는 점점 빨라지고 중간부터 끝까지는 점점 느리게 진행 transition-timing-function: ease-in-out
transition 일괄지정
회전하면서 크기가 커진다

counter 카운터


counter CSS가 관리하는 변수 CSS 규칙에 따라 값이 자동적으로 증가해서 문서에서의 위치에 따라 콘텐츠의 모양을 조절하는 기능을 제공
속성
content ::before와 ::after를 이용해서 특정 요소 앞/뒤에 콘텐츠 추가
counter-reset 하나 이상의 CSS 카운터를 생성/리셋
counter-increment CSS 변수의 값을 증가/감소
counter() 변수의 값을 콘텐츠로 추가시키는 함수
counters() 요약된 형태의 리스트를 만드는 경우 다른 수준의 중첩된 카운터 사이에 문자열을 추가하는 함수

가상 요소 ::before와 ::after를 이용하여특정 요소의 앞/뒤에 콘텐츠를 추가할 수 있도록 지정한다

--미디어쿼리


미디어쿼리 같은 스타일시트 내에서 서로 다른 미디어에 다른 스타일을 적용하기 위한 것 CSS3 미디어 쿼리
미디어 타입 + 미디어 타입의 특성
all → 모든 유형의 미디어 장치
print → 프린터
screen → 컴퓨터 스크린, 태블릿, 스마트폰 등
미디어특성
width(max- , min-) 출력 장치의 대상 표시 영역(뷰포트)의 폭
height(max- , min-) 출력 장치의 대상 표시 영역의 높이
device-width(max- , min-) 출력 장치가 제공하는 기본적인 폭
device-height(max- , min-) 출력 장치가 제공하는 기본적인 높이
orientation 화면의 가로 방향(landscape)과 세로 방향(portrait)
aspect-ratio(max-, min-) 화면 종횡비(width와 height의 비율)
device-aspect-ratio(max-, min-) 출력 장치 디스플레이의 물리적인 화면 비율(device-width와 device-height의 비율)
color(max-, min-) 출력 장치의 색 성분에 할당되는 비트 수
color-index(max-, min-) 출력 장치의 색상 룩업 테이블에 있는 전체 색상 수
resolution(max-, min-) 출력 장치의 해상도
scan TV 미디어의 화면 주사 방식(순차 주사 progressive, 비월주사 interlace)
grid 출력 장치가 그리드인지 비트맵인지를 표시

반응형웹


반응형 웹디자인 RWD: responsive web design 하나의 웹사이트가 다양한 크기와 특성을 갖는 장치에
가장 적합한 형태로 표시되도록 웹페이지를 제작하는 것
사용자가 어떤 장치를 사용하여 웹사이트에 접속하는가에 동적으로
반응하여 해당 장치에 최적화된 형태로 디자인을 변경할 수 있도록 설계
반응형 웹의 구현 → HTML과 CSS만 이용 관련 기술 -> 뷰포트 메타 태그 / 유동적 레이아웃 / 미디어 쿼리
특성
viewport 브라우저를 통해 사용자가 볼 수 있는 웹페이지 영역
→ 각 장치의 모니터와 뷰포트에 맞춰 웹페이지의 콘텐츠 폭의 조정이 필요 <meta name="viewport" content="width=device-width,initial-scale=1.0">
content="width=device-width: 페이지의 폭 = 장치의 스크린 폭 initial-scale=1.0 : 브라우저에 의해서 페이지가 처음 로드될 때의 초기 배율
유동적 레이아웃 그리드 개념에 기반하여 고정된 픽셀 크기가 아닌 화면에 따른 비율로 콘텐츠의 크기를 정의하는 것
그리드 개념 → 웹페이지를 여러 개의 열로 구분
브라우저 창의 크기에 따라 콘텐츠의 크기도 이에 비례하여 조정
보통 12개의 열을 가진 그리드 뷰 사용 출력 장치가 제공하는 기본적인 폭
device-height(max- , min-) 모든 콘텐츠의 크기는 열의 크기의 정수배로 표현
• 각 열의 폭 = 100% / 12열 = 8.33%
